## 1. 定位(一句话)

**灵剪 = 面向中文创作者与 AI 编程 Agent 的、"可审核 · 可复跑 · 可归档"的开源短视频生产工作台:把文本 / 链接 / 截图,变成可发抖音 / 小红书 / B站 / YouTube / Shorts 的中英文发布包。**

它不是剪辑器、不是一键玩具、不是 SaaS-only、不是 CLI-only。差异化护城河 = **三步人审门禁(文案·语音·画面)+ artifact 单一事实源 + 多平台发布包 + provider/引擎可插拔 + doctor**。渲染可换,内容理解→脚本→多平台打包→审核控制台这套"编排 + 治理"是核心。

## 2. 最终目标(分层,可判定)

**M1 产品级北极星(用户视角,不是工程 DoD):**
> 一个**不会剪辑**的中文创作者,粘贴一段中文文本(或一个链接 / 几张截图),**只做 3 次审核**(文案、语音、画面),在**几分钟内**得到一条**能播、中文不乱码、音画字同步、带封面和发布文案**的抖音 9:16 视频发布包——**全程零手工剪辑**。Agent 能经 CLI 走同一条链路;团队能查到"谁在什么版本上批准了什么"。

**北极星指标(主)+ 护栏(次):**
| 指标 | 定义 | M1 目标 |
|---|---|---|
| 主:审核通过率 | 进入渲染的项目中,一次三审通过(无返工)占比 | 观测基线,越高越好 |
| 主:输入→可发布时长 | 从新建到导出 release 包的中位耗时(真实 provider) | 单条 ≤ 数分钟(不含人审思考) |
| 护栏:渲染成功率 | 三审通过后 render 成功且 QA 无 hard fail 占比 | ≥ 95% |
| 护栏:中文可读率 | 导出视频帧内中文非空、无方块 | 100%(否则 hard fail) |
| 护栏:门禁有效性 | 未审 / 失效审批被 render 拒绝 | 100% |
| 护栏:release 纯净率 | release 包不含 mock provider | 100% |

**长期愿景(一句话,不进 M1):** 让"想法 → 多平台多语言可发布视频"的边际成本趋近于一次审核,并保持每一步可审计、可复跑、合规。

## 3. 受众与 JTBD(Persona)

| Persona | 核心诉求(JTBD) | 主要痛点 | M1 成功信号 |
|---|---|---|---|
| **中文知识/产品/教程创作者**(个人) | "我有内容,但不会剪辑,想快速产出能发的短视频" | 剪辑软件太重、配音/字幕/排版耗时、多平台尺寸重复劳动 | 粘文本→3 审→出抖音/小红书竖屏包,零剪辑 |
| **小团队 / MCN / 企业市场**(协作) | "要能审、能留痕、能批量、能多平台分发的生产流程" | 谁改了什么无记录、无法复跑、平台合规靠人肉 | 文案/语音/画面分角色审;每次审有 who+version;多平台一键导 |
| **AI Agent 使用者**(Codex/Claude Code) | "让 Agent 编排整条生产线,但关键处必须我拍板" | 一键工具黑箱、Agent 会跳过人审、结果不可控 | CLI 全 `--json`;生成即停等审;门禁不可绕 |
| **自部署团队**(隐私/合规) | "key、素材、审稿记录、导出包都留在我自己机器" | 云 SaaS 数据外泄、license 不清、模型权重合规风险 | 本地优先;doctor 指引;license_manifest;权重不捆绑 |

## 4. M1 真实场景(深化:触发→流程→输出→边缘)

1. **产品介绍短视频** — 触发:粘产品文案 / 官网链接 + 几张截图 → 流程:提取卖点+来源→脚本(hook/卖点/CTA)→配音→截图卡片画面→三审→渲染 → 输出:抖音/小红书/Shorts 9:16 包 → 边缘:截图含敏感信息(手机号/邮箱)需 QA 提示;卖点无来源需标"未验证"。
2. **教程/演示视频** — 触发:步骤文档 + 步骤截图 → 流程:分镜按步骤,截图卡片 + 旁白 → 输出:B站/YouTube 16:9 或教程 4:3 包 → 边缘:步骤过多导致超时长 → 建议拆分或提示。
3. **公众号/博客视频化** — 触发:URL / 正文 → 流程:trafilatura 提取正文→观点脚本→短视频 → 输出:小红书/Shorts → 边缘:动态页正文抽取过少(<阈值)→ 提示改用截图或粘正文;**默认不下载站点视频**。
4. **双语分发** — 触发:一份中文或英文稿 → 流程:bilingual=两个独立语言包(各自脚本/配音/字幕/平台文案)或单视频双行字幕 → 输出:zh-CN + en-US 包 → 边缘:双语非逐句直译,是各自平台化表达。

## 5. 约束矩阵(硬约束,贯穿设计)

| 类别 | 约束 |
|---|---|
| **范围** | M1 = 纵向主干真实可用;不做全宽度(6 引擎/20 provider/ASR/桌面全砍) |
| **架构** | Python core/CLI/providers/API/(M2)MCP;Next.js 仅 Web;Node 引擎子进程;core/provider 禁 import 引擎 SDK |
| **渲染** | M1 只走自研 `ffmpeg_card`,SCOPE FREEZE(静态卡片+帧内字幕+concat+finalizer,禁动画/转场) |
| **治理** | 三审门禁系统层强制 + 绑 artifact hash + 无 `--force`;mock 不能 release |
| **合规** | 主仓 Apache-2.0;AGPL/GPL 只研究不引入;Remotion 不默认/不捆绑;权重/字体不捆绑;URL 默认不下载他人视频;克隆默认关+授权 |
| **数据** | 文件 artifact = 唯一事实源;SQLite = 派生索引可 reindex;key 脱敏 |
| **性能** | M1 单项目同步;云 TTS 同步阻塞无后台队列;无批量并发(批量/异步 = M2) |
| **国际化** | 中文 UI/路径/字体零报错;CJK 字体缺失自动下载(OFL,不入仓);国内模型 base_url 可配 |
| **平台** | 5 平台 preset 纯配置层,单渲染路径,禁平台名 if;4:3 为教程/演示通用比例不绑平台原生规格 |
| **测试** | 默认 CI 离线,不依赖真实模型/网络/GPU/付费 key |

## 7. 反目标(明确不追求)

不做传统时间线剪辑器;不追一键黑箱(必须可审);不做视频搬运(不默认下载他人视频);不把 v1 做成大而全;不为 demo 好看牺牲真实可用;不复制第三方代码/prompt/UI;不锁单一模型/引擎;不在无授权下克隆他人声音。
